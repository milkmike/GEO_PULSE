# Спека: возвращение продуктовых страниц и слоёв данных

Дата: 2026-06-12 · Статус: утверждена пользователем · Исполнение: по фичам, атомарными коммитами в `main` (прод подтягивает автодеплоем)

## Цель

Вернуть в GEO PULSE v2 сильные стороны старого продукта (прозрачность источников,
конкретика со ссылками, страница миссии, данные ООН/торговли в досье) и добавить
первый слой по образцу World Monitor (аналитические центры). Ось анализа неизменна:
«страна ↔ Россия».

## Контекст (что уже есть)

- v2-фронт `web/` (Next.js): главная (карта+рейтинг+сигналы+бриф), `/signals`,
  `/country/[code]`, `/embed/[code]`.
- Старый фронт в ветке `origin/nextjs`: `sources/page.tsx`, `about/page.tsx`,
  `UNVotesChart.tsx`, `TradeChart.tsx`, `glossary.tsx` — порт-доноры.
- API: `/api/v1/sources` (полный список+статы), `/api/v1/countries/{code}/un-votes`,
  `/api/v1/countries/{code}/trade`, `/api/v2/*` (мир), `/api/v2/health/sources`.
- БД: `un_votes` (10 стран, до 2023), `trade_data` (10 стран, до 2024) — лоадеры
  `load_un_votes.py` (умеет 98 стран), `auto_trade_loader.py` (IMF DOTS бесплатно).
- `briefs.meta` jsonb — пустует, пригоден для цитат.
- Исследование WM: `docs/research/worldmonitor-layers.md`.

## Фича 1 — Сноски-цитаты в брифингах (подход A)

**Бэкенд** (`src/pipeline/briefs.py`):
- `gather_world_inputs` / `gather_country_inputs` нумеруют статьи сквозной нумерацией
  `[1..N]` и возвращают `citations: [{n, title, url, source, country}]`.
  В выборки заголовков добавить `ar.url` и имя источника.
- GDELT-сэмплы: цитируются только если в `article_samples` есть `url`; иначе идут
  в промпт без номера.
- Промпты (`WORLD_BRIEF_PROMPT`, `COUNTRY_BRIEF_PROMPT`): заголовки подаются как
  `[n] (страна, источник) заголовок`; инструкция «каждое фактическое утверждение
  помечай сноской [n]; не выдумывай номера».
- Постпроцессинг: regex `\[(\d+)\]` по сгенерированному тексту; сноски с номерами
  вне диапазона удаляются. В `briefs.meta.citations` сохраняется ПОЛНЫЙ список
  поданных в промпт статей, у каждой — флаг `used: bool` (номер встретился в тексте).
- `source_hash` не меняем (вход тот же + номера детерминированы).

**API** (`src/api/routes/world.py`): `/api/v2/brief` и `/api/v2/countries/{code}/brief`
дополнительно отдают `citations` из `meta`.

**Фронт** (`web/`): `Markdown.tsx` принимает проп `citations`; вхождения `[n]`
рендерятся как `<sup><a href target="_blank" title="{title} — {source}">n</a></sup>`.
Без citations поведение прежнее.

**Деградация**: модель не поставила сноски → бриф выглядит как сейчас. Ошибок нет.

**Тесты**: юнит на постпроцессинг (валидные/фантомные номера), юнит на сбор citations.

## Фича 2 — Блок «Главные новости дня» на главной

**API**: новый `GET /api/v2/headlines?hours=24&limit=20` — статьи c
`is_relevant=TRUE` за окно, сортировка `action_level DESC, reprint_count DESC,
published_at DESC`; поля: `title, url, source_name, tier, country_code,
country_name, published_at, sentiment, action_level`.

**Фронт**: нижний грид главной → 3 колонки: «Новости дня» | «Сигналы» | «Брифинг».
Строка: флаг, заголовок-ссылка (`target=_blank, rel=noopener`), источник, время,
цветовая точка сентимента. Автообновление вместе с остальной главной (120с).

**Тесты**: фикстурный тест эндпоинта (порядок сортировки, окно).

## Фича 3 — Страница `/sources` «Источники»

**Данные**: `/api/v1/sources` + `/api/v2/health/sources`, мёрж по `id` на фронте.
Если health-эндпоинт не отдаёт `source_id` — расширить его (минимальная правка).

**UI** (`web/app/sources/page.tsx`, порт логики старой страницы на стиль v2):
- Метрики: всего источников / активных / собрано статей / стран покрыто.
- Фильтры: страна, тир (7 значений с цветовыми бейджами), язык, живость
  (OK/STALE/DEAD), активность.
- Карточка источника: имя (ссылка на сайт), страна+флаг, тир-бейдж, язык,
  число статей, дата последней статьи, средний сентимент, статус живости.
- **Матрица языкового покрытия** (свёртываемая): строки — 99 стран, колонки —
  `ru / en / native`; ✓/— по наличию активного источника соответствующего языка;
  native определяется по `langs` страны (фича 4). Клик по ячейке = фильтр списка.
- Ссылка «источники» в шапке главной (рядом с «все сигналы»).

## Фича 4 — Языковые корзины

- `src/countries.py`: новое поле `langs: list[str]` (ISO-коды нативных языков)
  у всех 99 стран. Источник истины для колонки native в матрице.
- API: `/api/v2/meta` дополняется `langs` по странам.
- Кураторская фаза (отдельные PR после UI): для ~25–30 топ-стран (G20 + СНГ +
  горячие точки) добрать фиды в `src/collectors/sources_world.yaml` до нормы
  «≥1 ru, ≥1 en, ≥1 native». Каждый кандидат проверяется скриптом-валидатором
  (новый `scripts/validate_feed.py`: HTTP 200, валидный RSS/Atom, свежесть ≤30д).

## Фича 6 — Досье стран: ООН, торговля, договоры

**Данные**:
- `load_un_votes.py`: прогнать для всех стран реестра (поддерживает 98/99);
  добавить сервис в `docker-compose.yml` (loop-режим, 30 дней) — таблица
  `un_votes` вырастет с 10 до ~98 стран.
- `auto_trade_loader.py`: расширить с CIS-пар на все 99 стран (источник —
  IMF DOTS, бесплатный, без ключа); compose-сервис с месячным циклом.
- Таблицы `un_votes`/`trade_data` создаются лоадерами; добавить их DDL в
  `data/init.sql` и `scripts/migrations/` (идемпотентно) — по конвенции CLAUDE.md.

**API**: v2-прокси `/api/v2/countries/{code}/un-votes` и `/trade` (тонкие обёртки
над существующей v1-логикой, чтобы фронт жил только на v2), плюс новый
`/api/v2/countries/{code}/agreements`: события из `analysis` за 180 дней,
`event_type IN ('diplomatic','economic') AND action_level >= 3`, группировка по
`event_key`, в группе: max(action_level), даты, статьи (title, url, source).

**Фронт** (`web/app/country/[code]/page.tsx`): три новые секции —
«Голосования ООН» (порт UNVotesChart: 4 карточки + площадной график % совпадения,
линия 50%), «Торговля с Россией» (порт TradeChart: 4 карточки + бары экспорт/импорт
с линией оборота), «Договоры и намерения» (список событий-групп со ссылками).
Секции скрываются, если данных по стране нет.

## Фича 7 — Слой «Аналитические центры» (intel)

- `src/collectors/sources_world.yaml` (или отдельный `sources_analytics.yaml`):
  ~15–20 фидов tier=`analytics`, без привязки к стране (country_code = «—» или US/GB
  по штаб-квартире — решение: привязываем к стране ШК, как у обычных источников):
  Jamestown EDM, ISW, Carnegie (Politika), RUSI, CSIS, ECFR, Chatham House,
  Bellingcat, OCCRP, War on the Rocks, FPRI, Atlantic Council и т.п.
  Где нет RSS — приём WM: `https://news.google.com/rss/search?q=site:{domain}+when:7d`.
- Реестр источников: новые опциональные поля `state_affiliated: bool`,
  `propaganda_risk: low|medium|high` (в yaml и в таблице `sources` — миграция).
  Размечаем и существующие state-источники (TASS и др.).
- Пайплайн не меняется: фиды идут через обычный collect→analyze.
- API: `/api/v2/headlines` (фича 2) получает опциональные параметры
  `tier` и `country` — один эндпоинт обслуживает и «Новости дня», и аналитику.
- UI: страница `/analytics` — лента статей источников tier=analytics за 7 дней
  с фильтром по стране (тонкая обёртка над headlines-эндпоинтом); фильтр
  «аналитика» на странице `/sources`; в «Новостях дня» бейдж тира.

## Не входит (будущие спеки, добавить в ROADMAP)

Финансовый «радар изоляции РФ» (композит сигналов), энергетический слой
(CREA, Urals-дисконт, карта труб), технологическая развязка (экспортный контроль,
ИИ-активы), санкционный слой (OFAC/OpenSanctions), prediction markets, Telegram-OSINT.

## Порядок реализации

1 → 2 → 3 → 4 → 5 → 6 → 7 (нумерация фич выше). Каждая фича — отдельный коммит
(или серия), пуш в `main`, автодеплой, проверка на проде. Термометр v1 и API v1
не ломаем (конвенция CLAUDE.md). Новые таблицы — и в migrations, и в init.sql.

## Тестирование

- Юниты: постпроцессинг сносок, citations-сборка, headlines-эндпоинт,
  agreements-группировка (фикстуры по схеме из CLAUDE.md, feedparser-заглушка).
- Ручная проверка на проде после каждого автодеплоя (эндпоинты + страницы).
- Валидатор фидов перед добавлением каждого нового источника.
